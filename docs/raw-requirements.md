# 협력형 챌린지 앱 - 초기 요구사항

## 기능적 요구사항
- 챌린지 진행 상황 시각적 정리
- 개별 챌린지 페이지 내 인증 및 피드백 기능
- 달력 페이지를 통한 진행률 표시

- 인증 사진 아래 일기 및 피드백 공간
- 챌린지 완료 시 디자인 변화 아이디어
- 병원 내 앱 테스트 및 피드백 수렴

- 개인 챌린지 앱의 동기 부여 부족
- 함께하는 챌린지의 으쌰으쌰 효과
- 다수 참여 시 성취감 증대

- 내 페이지에서는 내가 하고 있는 챌린지들이 정리되어야 함
- 카테고리 하나를 누르면 다른 사람들과 함께 있는 공간으로 들어가게 됨
- 사람들이 모이는 공간은 귀엽게 표현
  - 예: 여름에는 잎사귀, 봄에는 꽃
- 그 공간에 들어가면 달력 페이지가 보임
- 달력에서 사람별로 누르면 해당 챌린지의 인증 내역이 보임

## 기술적 요구사항
- 프론트엔드는 Flutter로 개발
- iOS / Android 동시 지원
- 백엔드는 Python으로 개발

## 현재 내가 중요하게 보는 포인트
- 동기부여가 되는 협력형 경험
- 귀엽고 계절감 있는 시각화
- 인증 사진 + 일기 + 피드백 구조
- 병원 파일럿 테스트 가능성
