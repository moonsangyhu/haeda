# 캐릭터 커스터마이징 & 재화 시스템 기획서

> **Status**: P0 (병원 파일럿 MVP 포함)
> **Created**: 2026-04-10
> **Version**: 1.0

## 1. 개요

Haeda 앱의 핵심 셀링 포인트로 **캐릭터 꾸미기 + 재화 시스템**을 도입한다.

**핵심 루프**: 챌린지 인증 → 코인 획득 → 아이템 구매 → 캐릭터 꾸미기 → 챌린지 방에서 자랑 → 더 열심히 인증

- **비주얼**: 2D 픽셀아트 (레이어 기반 조합)
- **재화**: 인증 기반 + 연속 보너스
- **동기부여**: 챌린지 방에서 서로의 캐릭터를 보며 "나도 저렇게 꾸미고 싶다"는 욕구 자극

## 2. 유저 스토리

### 2.1 재화 획득

| ID | As a... | I want to... | So that... | 수락 기준 |
|----|---------|-------------|------------|-----------|
| US-C01 | 사용자 | 일일 인증 완료 시 코인을 받고 싶다 | 캐릭터를 꾸밀 재화를 모을 수 있다 | 인증 제출 성공 시 +10 코인, 토스트로 획득 알림 표시 |
| US-C02 | 사용자 | 전원 인증 달성 시 보너스를 받고 싶다 | 팀 전체가 협력할 동기가 생긴다 | DayCompletion 생성 시 참여 멤버 전원 +20 보너스 코인 |
| US-C03 | 사용자 | 챌린지 완주 시 큰 보상을 받고 싶다 | 챌린지를 끝까지 완주할 동기가 있다 | 챌린지 완료 + 참여 시 +100 코인 |
| US-C04 | 사용자 | 연속 인증 시 추가 보너스를 받고 싶다 | 매일 꾸준히 인증할 동기가 있다 | 3일 연속 +15, 7일 연속 +50 (같은 챌린지 내) |
| US-C05 | 사용자 | 앱 접속(출석) 시 소량의 코인을 받고 싶다 | 매일 앱을 열어볼 이유가 있다 | 일 1회 앱 접속 시 +5 코인 |
| US-C06 | 사용자 | 현재 보유 코인을 항상 확인하고 싶다 | 구매 계획을 세울 수 있다 | 상단 바 또는 내 방 탭에 코인 잔액 상시 표시 |

## 3. 도메인 모델 (신규 엔티티)

### 3.1 CoinBalance (재화 잔액)

| 필드 | 타입 | 설명 |
|------|------|------|
| user_id | UUID (PK, FK → User) | 사용자 |
| balance | Integer (>= 0) | 현재 코인 잔액 |
| updated_at | DateTime | 마지막 변경 시각 |

### 3.2 CoinTransaction (재화 내역)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID (PK) | |
| user_id | UUID (FK → User) | 사용자 |
| amount | Integer | 양수=획득, 음수=소비 |
| type | Enum | VERIFICATION, ALL_COMPLETED, CHALLENGE_DONE, STREAK_3, STREAK_7, DAILY_LOGIN, PURCHASE |
| reference_id | UUID? | 관련 챌린지ID 또는 아이템ID |
| created_at | DateTime | 거래 시각 |

### 3.3 Item (아이템 카탈로그)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID (PK) | |
| name | String | 아이템 이름 |
| category | Enum | HAT, TOP, BOTTOM, SHOES, ACCESSORY |
| price | Integer | 코인 가격 |
| rarity | Enum | COMMON, RARE, EPIC |
| asset_key | String | 에셋 파일 경로 키 |
| is_active | Boolean | 상점 진열 여부 |
| sort_order | Integer | 정렬 순서 |
| created_at | DateTime | |

### 3.4 UserItem (인벤토리)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID (PK) | |
| user_id | UUID (FK → User) | 소유자 |
| item_id | UUID (FK → Item) | 아이템 |
| purchased_at | DateTime | 구매 시각 |

- UNIQUE(user_id, item_id)

### 3.5 CharacterEquip (캐릭터 착용 상태)

| 필드 | 타입 | 설명 |
|------|------|------|
| user_id | UUID (PK, FK → User) | 사용자 |
| hat_item_id | UUID? (FK → Item) | 착용 모자 |
| top_item_id | UUID? (FK → Item) | 착용 상의 |
| bottom_item_id | UUID? (FK → Item) | 착용 하의 |
| shoes_item_id | UUID? (FK → Item) | 착용 신발 |
| accessory_item_id | UUID? (FK → Item) | 착용 액세서리 |
| updated_at | DateTime | 마지막 변경 |

### 비즈니스 룰

- 각 슬롯에는 해당 카테고리 아이템만 착용 가능 (hat_item_id에는 HAT 카테고리만)
- 보유하지 않은 아이템은 착용 불가
- 코인 잔액이 부족하면 구매 불가 (INSUFFICIENT_COINS 에러)
- 이미 보유한 아이템은 재구매 불가 (ALREADY_OWNED 에러)
- 코인 지급은 서버 내부 이벤트로만 처리 (클라이언트 직접 호출 불가)

## 4. API 엔드포인트

### 4.1 코인 API

#### GET /me/coins
내 코인 잔액 조회.

**Response 200:**
```json
{
  "data": {
    "balance": 1250
  }
}
```

#### GET /me/coins/transactions
코인 획득/소비 내역 (최신순, 페이지네이션).

**Query Params:** `cursor`, `limit` (default 20, max 50)

**Response 200:**
```json
{
  "data": {
    "items": [
      {
        "id": "uuid",
        "amount": 10,
        "type": "VERIFICATION",
        "reference_id": "challenge-uuid",
        "created_at": "2026-04-10T09:00:00Z"
      }
    ],
    "next_cursor": "uuid-or-null"
  }
}
```

### 4.2 상점 API

#### GET /shop/items
상점 아이템 목록.

**Query Params:** `category` (optional: HAT, TOP, BOTTOM, SHOES, ACCESSORY)

**Response 200:**
```json
{
  "data": [
    {
      "id": "uuid",
      "name": "신사모자",
      "category": "HAT",
      "price": 50,
      "rarity": "COMMON",
      "asset_key": "hat/hat_gentleman.png",
      "is_owned": false
    }
  ]
}
```

#### POST /shop/items/{item_id}/purchase
아이템 구매.

**Response 200:**
```json
{
  "data": {
    "item_id": "uuid",
    "remaining_balance": 1200
  }
}
```

**Errors:**
- `INSUFFICIENT_COINS` — 코인 부족
- `ALREADY_OWNED` — 이미 보유
- `ITEM_NOT_FOUND` — 아이템 없음

### 4.3 인벤토리 & 캐릭터 API

#### GET /me/items
내 보유 아이템 목록.

**Response 200:**
```json
{
  "data": [
    {
      "id": "user-item-uuid",
      "item": {
        "id": "uuid",
        "name": "신사모자",
        "category": "HAT",
        "rarity": "COMMON",
        "asset_key": "hat/hat_gentleman.png"
      },
      "purchased_at": "2026-04-10T09:00:00Z"
    }
  ]
}
```

#### GET /me/character
내 캐릭터 착용 상태.

**Response 200:**
```json
{
  "data": {
    "hat": { "id": "uuid", "name": "신사모자", "asset_key": "hat/hat_gentleman.png", "rarity": "COMMON" },
    "top": null,
    "bottom": null,
    "shoes": null,
    "accessory": null
  }
}
```

#### PUT /me/character
캐릭터 착용 상태 변경.

**Request:**
```json
{
  "hat_item_id": "uuid-or-null",
  "top_item_id": "uuid-or-null",
  "bottom_item_id": null,
  "shoes_item_id": null,
  "accessory_item_id": null
}
```

**Errors:**
- `ITEM_NOT_OWNED` — 보유하지 않은 아이템 착용 시도
- `INVALID_CATEGORY` — 슬롯과 카테고리 불일치

#### GET /users/{user_id}/character
다른 유저 캐릭터 조회 (챌린지 방 멤버용). 응답 형식은 `/me/character`와 동일.

## 5. 재화 경제 설계

### 5.1 코인 획득 테이블

| 이벤트 | 코인 | 조건 | 빈도 |
|--------|------|------|------|
| 일일 인증 완료 | +10 | 인증 제출 성공 | 챌린지당 1일 1회 |
| 전원 인증 달성 | +20 | DayCompletion 생성 시 | 참여자 전원에게 |
| 챌린지 완주 | +100 | 챌린지 종료 + 참여 | 챌린지당 1회 |
| 3일 연속 인증 | +15 | 같은 챌린지 내 3일 연속 | 달성 시마다 |
| 7일 연속 인증 | +50 | 같은 챌린지 내 7일 연속 | 달성 시마다 |
| 일일 출석 | +5 | 앱 접속 | 1일 1회 |

### 5.2 아이템 가격 가이드

| 등급 | 가격 범위 | 획득 소요 기간 | 예시 |
|------|-----------|---------------|------|
| COMMON | 30~80 코인 | 3~5일 | 기본 티셔츠, 슬리퍼, 심플 모자 |
| RARE | 100~200 코인 | 1~2주 | 후드티, 스니커즈, 선글라스 |
| EPIC | 300~500 코인 | 2~4주 | 왕관, 턱시도, 날개 액세서리 |

### 5.3 경제 시뮬레이션 (1개 챌린지 참여 기준)

| 기간 | 최소 수입 (인증만) | 최대 수입 (전원+스트릭) |
|------|-------------------|----------------------|
| 1일 | 15 (인증10+출석5) | 55 (인증10+전원20+출석5+스트릭20평균) |
| 1주 | 105 | ~330 |
| 4주 | 420 | ~1,420 |

> 목표: 열심히 하면 1주에 COMMON 2~3개, 4주면 EPIC 2~3개 구매 가능.

## 7. 에셋 전략

### 7.1 픽셀아트 에셋 구조

```
app/assets/character/
├── base/
│   ├── body.png          # 기본 캐릭터 바디 (64x64)
│   └── face.png          # 기본 얼굴
├── hat/
│   ├── hat_cap.png
│   ├── hat_gentleman.png
│   ├── hat_crown.png
│   └── ...
├── top/
│   ├── top_tshirt_white.png
│   ├── top_hoodie_blue.png
│   └── ...
├── bottom/
│   ├── bottom_jeans.png
│   ├── bottom_shorts.png
│   └── ...
├── shoes/
│   ├── shoes_sneakers.png
│   ├── shoes_slippers.png
│   └── ...
└── accessory/
    ├── acc_sunglasses.png
    ├── acc_wings.png
    └── ...
```

### 7.2 기술 스펙

- **포맷**: PNG (투명 배경)
- **기본 사이즈**: 64x64 px (디스플레이 시 nearest-neighbor 스케일업으로 픽셀 느낌 유지)
- **레이어 순서** (아래→위): body → bottom → shoes → top → hat → accessory
- **렌더링**: Flutter `Stack` + `Image` 위젯으로 레이어 합성
- **썸네일**: 챌린지 멤버 목록에서는 40x40 dp로 축소 표시

### 7.3 초기 아이템 세트 (MVP)

| 카테고리 | COMMON (3개) | RARE (2개) | EPIC (1개) |
|----------|-------------|-----------|-----------|
| 모자 | 캡모자, 비니, 머리띠 | 페도라, 베레모 | 왕관 |
| 상의 | 흰티, 줄무늬티, 민소매 | 후드티, 가디건 | 턱시도 |
| 하의 | 청바지, 반바지, 면바지 | 치마, 카고바지 | 황금바지 |
| 신발 | 슬리퍼, 운동화, 샌들 | 부츠, 하이탑 | 날개신발 |
| 액세서리 | 시계, 가방, 스카프 | 선글라스, 이어폰 | 천사날개 |

**총 30개 아이템** (카테고리당 6개)

## 8. 구현 슬라이스 계획

### Phase 1: 코인 시스템 (백엔드)
- DB 마이그레이션: `CoinBalance`, `CoinTransaction` 테이블
- 코인 서비스: 지급/차감/조회 로직
- 인증 완료 이벤트에 코인 지급 훅
- 연속 인증 스트릭 계산 로직
- API: `GET /me/coins`, `GET /me/coins/transactions`

### Phase 2: 아이템 & 상점 (백엔드)
- DB 마이그레이션: `Item`, `UserItem`, `CharacterEquip` 테이블
- 시드 데이터: 초기 아이템 30개
- API: 상점 목록, 구매, 인벤토리, 캐릭터 조회/변경
- 기존 API 확장 (calendar members에 character 추가)

### Phase 3: 캐릭터 렌더링 (프론트)
- 픽셀아트 에셋 제작/배치
- `CharacterAvatar` 위젯 (레이어 스택 렌더링)
- 캐릭터 상태 Provider (내 캐릭터 + 타인 캐릭터)

### Phase 4: 내 방 탭 (프론트)
- 하단 탭 4탭으로 확장
- MyRoom 화면: 캐릭터 표시 + 인벤토리 + 착용 UX
- 코인 잔액/내역 표시

### Phase 5: 상점 화면 (프론트)
- 상점 화면: 카테고리 탭 + 아이템 그리드 + 미리보기
- 구매 흐름 (확인 다이얼로그 → 결과)

### Phase 6: 챌린지 방 개선 (프론트)
- 멤버 목록 캐릭터 아바타로 교체
- 캐릭터 탭 → 상세 바텀시트
- 인증 완료 시 코인 획득 애니메이션

### Phase 7: 통합 & 폴리시
- 출석 코인 지급 로직
- 코인 부족 시 안내 UX
- 에지 케이스 처리
