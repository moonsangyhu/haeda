> 버전: 0.1 (MVP)
> 최종 수정: 2026-04-04

---

## 1. 제품 개요

### 1.1 문제
개인 챌린지 앱은 혼자 하다 보면 동기부여가 부족해 쉽게 포기하게 된다.

### 1.2 솔루션
**참여자 전원이 함께 인증해야 예쁜 계절 아이콘이 완성되는** 협력형 챌린지 앱.
개인이 인증하면 달력에 프로필 썸네일이 표시되고, 해당 날짜에 참여자 전원이 인증을 완료하면 계절감 있는 아이콘(봄: 꽃, 여름: 나뭇잎, 가을: 단풍, 겨울: 눈꽃)으로 바뀐다. "예쁜 달력을 함께 만들자"는 동기가 참여자 간 격려를 자연스럽게 유도한다.

### 1.3 핵심 가치
- **함께하는 으쌰으쌰** — 협력형 인증으로 상호 격려
- **귀엽고 계절감 있는 시각화** — 전원 인증 시 보상이 되는 시각적 즐거움
- **가벼운 일상 기록** — 인증 사진 + 일기 + 댓글로 소통

### 1.4 대상 사용자
- 1차: 병원 파일럿 (환자 + 직원, 대상 미확정)
- 2차: 일반 사용자 확장

### 1.5 기술 스택
| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | Flutter (iOS / Android) |
| 백엔드 | Python FastAPI |
| 데이터베이스 | PostgreSQL |
| 인증 | 카카오 OAuth |
| 푸시 알림 | FCM (Firebase Cloud Messaging) |

## 2. MVP 기능 목록

### 2.1 인증/사용자
| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 카카오 소셜 로그인 | 카카오 OAuth를 통한 회원가입/로그인 |
| F-02 | 프로필 설정 | 닉네임, 프로필 사진 설정 |

### 2.2 챌린지
| ID | 기능 | 설명 |
|----|------|------|
| F-03 | 챌린지 생성 | 제목, 설명, 카테고리(자유 입력), 기간(시작일~종료일), 인증 빈도(매일/주N회), 사진 필수 여부, 공개 여부 설정 |
| F-04 | 초대 링크/코드 | 챌린지 생성 시 초대 코드 자동 생성, 링크 공유 |
| F-05 | 공개 챌린지 탐색 | 공개 챌린지 목록 조회 및 참여 |
| F-06 | 챌린지 참여 | 초대 코드 또는 탐색을 통한 참여 (인원 제한 없음) |
| F-07 | 챌린지 완료 | 기간 종료 시 자동 완료, 개인별 달성률 표시, 완료 배지 부여 |

### 2.3 내 페이지
| ID | 기능 | 설명 |
|----|------|------|
| F-08 | 내 챌린지 목록 | 참여 중인 챌린지 카드 목록 + 각 챌린지 달성률 요약 |

### 2.4 챌린지 공간 (달력 뷰)
| ID | 기능 | 설명 |
|----|------|------|
| F-09 | 달력 기반 협력 뷰 | 월별 달력에 날짜별 인증 현황 표시 |
| F-10 | 개인 인증 표시 | 인증한 사람의 프로필 사진 썸네일이 해당 날짜에 표시 |
| F-11 | 전원 인증 계절 아이콘 | 해당 날짜에 참여자 전원이 인증하면 계절 아이콘으로 변환 |
| F-12 | 사람별 인증 내역 조회 | 달력에서 날짜 → 사람 선택 시 인증 상세(사진+일기) 조회 |

### 2.5 인증 & 피드백
| ID | 기능 | 설명 |
|----|------|------|
| F-13 | 인증 제출 | 사진(필수/선택, 챌린지 설정에 따름) + 일기(텍스트) 제출 |
| F-14 | 참여자 간 댓글 | 인증 내역에 대한 댓글/응원 작성 |

### 2.6 알림
| ID | 기능 | 설명 |
|----|------|------|
| F-15 | 인증 리마인더 | 아직 인증하지 않은 날에 리마인더 푸시 |
| F-16 | 팀원 인증 알림 | 같은 챌린지 참여자가 인증했을 때 알림 |

### 2.7 파일럿 피드백
| ID | 기능 | 설명 |
|----|------|------|
| F-17 | 사용 데이터 수집 | 접속 빈도, 인증율, 댓글 수 등 핵심 지표 트래킹 |

## 3. MVP 제외 범위

| 기능 | 제외 사유 |
|------|-----------|
| 관리자 대시보드 | MVP는 사용자 중심, 관리 기능은 이후 |
| 달성률 기반 랭킹/등급 | 경쟁보다 협력에 집중 |
| 앱 내 설문 | 사용 데이터 분석으로 대체 |
| 채팅/실시간 메시지 | 댓글로 충분, 복잡도 높음 |
| 챌린지 템플릿/추천 | 사용자 자유 생성으로 시작 |
| Apple 로그인 | iOS 심사 시 추가 예정 |

## 4. 미결정 사항

| # | 항목 | 영향 | 우선순위 |
|---|------|------|----------|
| 1 | 병원 파일럿 대상 확정 (환자 vs 직원) | UX 톤, 온보딩 설계 | 높음 |
| 2 | 인증 빈도 옵션 구체화 (주N회의 N 범위, 하루 복수 인증 여부) | 챌린지 생성 UI, 달성률 계산 로직 | 중간 |
| 3 | 완료 배지 디자인 규칙 (달성률 구간별 vs 단일) | 배지 에셋 준비, 동기부여 설계 | 중간 |
| 4 | 사진 저장소 선택 (S3 / Firebase Storage / R2) | 인프라 비용, 업로드 구현 | 중간 |
| 5 | 푸시 알림 타이밍 구체화 | 사용자 경험, 알림 피로도 | 낮음 |
| 6 | 챌린지 삭제/탈퇴 정책 | 데이터 정합성 | 낮음 |
| 7 | 데이터 분석 수집 이벤트 목록 | 파일럿 피드백 품질 | 낮음 |
